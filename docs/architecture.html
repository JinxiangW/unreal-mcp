Unreal Harness 架构草案

目标不是继续堆一个“大而全”的 MCP，而是构建一个总控层负责路由和统一错误模型， 再把具体执行拆到按问题域划分的子 harness。Scene / Asset / Material Asset 优先改为 UE Python 后端， Material Graph、Blueprint Graph 和部分 Niagara / 底层桥接继续保留在 C++ 插件层。

总控 / 路由 UE Python 子域 C++ 插件子域 共享协议与验证 待清理旧实现
Top Layer
总控入口

unreal-orchestrator

统一接收 AI 请求，做任务分类、preflight、子 harness 路由、结果归一化、错误分层和最终验证。

  • 选择 scene / asset / material / material-graph / niagara / blueprint / diagnostics
  • 统一 capability 探测
  • 统一返回格式与错误码
Domain Harnesses
UE Python

unreal-scene

场景布光、后处理、Actor 摆放、Level / Viewport 操作。

  • 高层命令优先
  • 避免直接裸写 properties
UE Python

unreal-asset

通用资产 CRUD、批量修改、导入、路径查询。

  • AssetTools / EditorAssetLibrary
  • 适合稳定通用资产流
UE Python

unreal-material

材质资产、材质实例、参数设置和基础属性管理。

  • 适合 UE Python 稳定落地
  • 不承担复杂 graph editing
C++ 主 / 混合

unreal-niagara

Niagara 图和 emitter 操作。优先保留混合模式，后续再收缩接口。

C++ 主 / 混合

unreal-material-graph

材质节点、连线、图构建、复杂图重构。单独作为 graph 编辑域处理。

UE Python

unreal-blueprint-info

蓝图读取、结构分析、内容快照、基础信息聚合。

C++ 主

unreal-blueprint-graph

节点、连线、变量、函数、事件图编辑。完整图编辑能力保留在插件层。

跨域

unreal-diagnostics

日志检查、能力探测、协议排障、回归验证、风险提示。

Execution Backends
默认执行层

UE Python Runtime

通过 PythonScriptPlugin 调用 Editor API，承担大多数高层编辑器语义。

  • 场景编辑更稳
  • 资产/材质工作流更贴近编辑器语义
  • 减少 Actor/Component/Struct 手工映射
保留能力层

C++ Plugin Backend

保留协议入口、命令注册、Blueprint Graph 编辑、部分 Niagara / 底层补齐能力。

  • TCP / bridge / registry
  • 复杂图编辑和反射缺口
  • 性能敏感或 Python 缺失的能力
Shared Infrastructure

Capability Registry

单一真实能力源。总控和子 harness 都从这里读命令面，不再手写多份命令表。

Error Model

统一 transport / protocol / capability / validation / runtime / partial-apply 错误层级。

Post-Apply Verification

修改后强制回读关键状态，必要时补截图，而不是只返回 success。

Migration Flow
Phase 1 建立 `unreal-orchestrator` 和问题域分类，保持现有功能还能跑。
Phase 2 优先重写 `scene / asset / material` 到 UE Python 后端。
Phase 3 单独抽出 `material-graph`，不要与普通材质资产流混合。
Phase 4 清理旧的 C++ 高层属性路由，只保留必要底层能力和复杂图编辑。
Phase 5 收敛 Niagara 和 Blueprint Graph 的边界，统一 capability registry 和错误模型。

原则：对外维持一个“总体可控”的入口，对内按域拆分实现；高层编辑语义优先落 UE Python，复杂图编辑和协议层留在 C++ 插件。

Legacy Removed

旧式高层属性路由

尤其是把场景编辑业务压成 `set_actor_properties(properties={...})` 的路径，应逐步收缩。

多处手工同步命令表

Python wrapper、bridge、具体 handler 三处手动同步，是当前最容易漂移的源头。
